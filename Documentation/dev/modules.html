Window Modules

Each module is a ModuleBase subclass rendering one ImGui window. GetName() is both the window's config key (in UiConfig) and its /myui <name> toggle name. Modules read all settings live from m_ctx.UI each frame and own only transient runtime state. The table is in registration / render order.

Notes

  • Default visibility: only HUD and Toolbar are seeded visible; every other window defaults hidden on fresh characters. See UiConfig::SeedDefaults.
  • Transient windows (BigBag, Inventory, iTrack, Settings, AA) never persist visibility and always start hidden — they are summoned from the Toolbar. See Settings & UiConfig.
  • GroupModule owns two window keys: Group and Raid are separate UiConfig entries (each its own scaling/visibility/bars; Raid reuses Group's HP/Mana/End/Pet roles). Both group rows and raid tiles render through the same DrawMemberRow path, so each shows 3D member distance (Distance3DToSpawn) plus the sit / moving (teal ICON_MD_DIRECTIONS_RUN) / standing and LOS status icons.
  • Peer-consuming modules: XpBarsModule, GroupModule, and BuffsModule read ActorManager::Peers(); the rest are local-only.
  • Spell gem recast overlay: DrawRadialCooldown (SpellsModule.cpp:57, called :418) draws a circular clock-sweep shadow over the square gem icon while on recast. The dark wedge is sized to remaining = (SpellGemETA[i] - now) / spell->RecastTime and its leading edge sweeps clockwise from 12 o'clock, clearing to nothing at ready. Rendered as a triangle fan from the icon center (anti-aliased fill toggled off across the fan to avoid alpha seams), radius = icon half-size + 3px, color kCooldownShadow. The numeric countdown still draws on top.
  • Player name in combat: PlayerModule draws the name with myui::DrawScrollingGradientText (red→white scrolling gradient via the coverage alpha-mask; see Bars & Widgets) when CharSnapshot::inCombat is set (PlayerModule.cpp:150), independent of the CombatPulse border-flash flag.
  • BigBag sort key: BigBagModule::SortItems chains the SortType/SortName/SortStack flags in priority type→name→stack. "By type" compares the descriptive item-type string from myui::ItemTypeName (InventoryData.cpp:943 — szItemClasses[ItemClass], szCombineTypes[Combine] for containers, "Book" for books), not the raw ItemDefinition::Type enum (which only separates NORMAL/PACK/BOOK and collapsed all normal items into one group). Matches the original MyUI Lua. ItemTypeName was extended for this and also backs the Details-tab Type label.

SpellPicker — helper, not a window

SpellPicker (SpellPicker.h:12) is a reusable spell-selection dialog (category tree + search) embedded by spell-related modules. Key methods: DrawSpellPicker (SpellPicker.cpp:164), DrawSpellTree (:10), InitializeSpells (:65), FilterSpells (:136), SetOpen / ClearSelection. It holds SpellData records (SpellPicker.h:15) and is not registered with the ModuleManager.
