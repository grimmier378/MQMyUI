Bars & Widgets

MQMyUI ships a custom-drawn widget set that gives it a consistent animated look: a fully styled progress bar (BarEngine) plus a family of imanim-driven controls in Core/Widgets.{h,cpp} — toggle, pill tabs/lists, accordion tree, collapsing headers, glowing sliders, combos, checkbox, drop-in styled buttons, a cursor-anchored input popup and a click-to-edit field. They render through the ImGui draw list and read colors live from the active theme. The bar is driven by UiConfig data (per-window, no code changes).

Shared across plugins. Core/Widgets.{h,cpp} is reused verbatim by the sibling MQMy* plugins, so it stays dependency-light (ImGui + imanim + mq/imgui/AlphaMask.h) and theme-neutral. The full reference lives in the KB: D:\CLAUDE\MQMyUI\KnowledgeBase\styled-widgets.md (the source of truth shared by MQMyDPS/MQMyChat). This page is the developer-facing overview.

The bar engine (Core/BarEngine.h/.cpp)

Two public entry points consume a BarStyle and draw the whole bar:

namespace myui {
    // Lays out the bar at the current ImGui cursor position.
    void DrawStyledBar(const char* id, float pct, const BarStyle& style,
                       int curValue = -1, int maxValue = -1);

    // Draws at an explicit screen rect given two corner points (top-left p0,
    // bottom-right p1) — use for overlays or anchored positions.
    void DrawStyledBarRect(const char* id, float pct, const BarStyle& style,
                           const ImVec2& p0, const ImVec2& p1,
                           int curValue = -1, int maxValue = -1);
}

pct is 0..100 (clamped internally before use). curValue/maxValue are optional and only used when the style's text overlay shows raw or cur·max values. The id keys the per-bar tween state, so each bar animates independently.

What it renders, in order

  1. Tween — eases the displayed value toward the target (keyed by id).
  2. Background — bgColor rounded rect.
  3. Fill — a single AddRectFilledMultiColor call with 4 corner colors, followed by a coverage-mask pass that clips the corners to produce the rounded look. No strip-slicing is used.
  4. Ticks, shimmer, glow, border.
  5. Text overlay — none / value / percent / cur·max, with a per-bar text scale.
Gradient + rounded corners. The fill uses AddRectFilledMultiColor with 4 independently computed corner colors (derived from gradientDir and the low/high fill colors), then a second coverage-mask pass clips the quad corners to the bar's rounding radius. This avoids strip-slicing while still supporting rounded corners with diagonal gradients.

Defaults are seeded from the original hardcoded look with all effects off, so nothing changes visually until a user opts in. Modules fetch a style with m_ctx.UI->Bar(GetName(), "HP") and pass it straight to DrawStyledBar.

There is no Tier 4 (no image-mask overlay) and no OKLAB color space — color interpolation is plain RGB lerp. The legacy DrawBar/DrawBarPct/DrawVertBar in ColorUtil.h remain as simple shims.
Internal helpers — not public API. DrawMaskedFill and DrawStencilMaskedFill live in an anonymous (file-local) namespace inside BarEngine.cpp. They are implementation details of the fill/coverage-mask pass and are not part of the myui:: public interface.

Gradient-masked text (DrawScrollingGradientText)

A public BarEngine helper (BarEngine.h:17, defined BarEngine.cpp:340) that fills the glyphs of a string with an animated gradient by reusing the coverage alpha-mask — the text itself is the mask:

void DrawScrollingGradientText(ImDrawList* dl, const ImVec2& pos, const char* text,
                              const ImVec4& colorA, const ImVec4& colorB);
  • CreateCoverageMaskLayer(dl, p0, p1) over the text's bounding box, then dl->AddText(p0, IM_COL32_WHITE, text) writes the glyph coverage into the framebuffer alpha.
  • Between BeginCoverageMaskedDraw / EndCoverageMaskedDraw the box is filled with ~32 stacked AddRectFilledMultiColor bands. Because coverage masking takes opacity from the mask and treats content as full-alpha color, the opaque gradient renders as gradient-filled, anti-aliased glyphs.
  • Band color is a triangle wave (colorA→colorB→colorA, one cycle = text height) offset by phase = fmod(GetTime()*scrollSpeed, 1) with a -phase term, so the band scrolls downward and loops seamlessly. scrollSpeed and period are local constants.
  • Reserves no layout space — the caller advances the cursor (e.g. ImGui::Dummy(CalcTextSize(text))).
First consumer: PlayerModule draws the player name with this (red→white) when CharSnapshot::inCombat is set (PlayerModule.cpp:152), else plain ImGui::Text. The helper is generic and intended for reuse on other text. Unlike DrawMaskedFill, this one is part of the myui:: public interface.

imanim integration (the engine behind the controls)

All motion is driven by imanim (iam_*), compiled into MQ core's ImGui. The plugin owns an imanim context and makes it current around its render pass in OnUpdateImGui; it must not call iam_update_begin_frame() (the host drives the clock — calling it doubles the speed) and must iam_context_destroy() on unload (a dangling context crashes the host). Tweens are keyed by ImGui::GetID(label) + a channel hash, so each instance animates independently. Full wiring + gotchas: see the KB.

Animation master switch. SetAnimationsEnabled(bool) / AnimationsEnabled() gate the whole set. Every iam_tween_* call in the file goes through file-local AnimFloat/AnimColor wrappers that snap to the target (no motion) when off; effect helpers no-op. It's pushed once per frame in MQMyUI.cpp:OnUpdateImGui from the per-char Flag("Global","AnimatedWidgets") (Settings → General toggle). Bars are not gated — they keep their own per-bar shimmer/glow settings.

The toggle switch (DrawToggle)

A pill-track toggle used for every boolean across the UI:

namespace myui {
    bool DrawToggle(const char* id, bool* value,
                    int flags = kUseGlobalToggleFlags,
                    ImVec2 size = ImVec2(0,0));   // returns true on change

    void SetGlobalToggleStyle(int flags, ImVec2 size = ImVec2(0,0));
}

Passing kUseGlobalToggleFlags (-1) makes a toggle inherit the global style set by SetGlobalToggleStyle — which the Settings → Toggles tab drives — so one place controls the look of every switch. Motion (all imanim): the thumb slides with an out_back overshoot, the track crossfades on↔off in OKLAB, the glow/pulse fade in/out through a hover-intensity tween, and the knob rocks toward its direction of travel as it settles. With ToggleFlags_StarKnob the off state draws a moon as a single alpha-masked crescent (mq::imgui stencil Subtract, not stacked circles) and the on state a star. The switch sizes to the label's text height (CalcTextSize(label).y), not the padded frame.

Styled buttons (StyledButton / StyledSmallButton)

Drop-in replacements for ImGui::Button/SmallButton (used project-wide); same sizing and release-activation semantics:

bool StyledButton(const char* label, ImVec2 size = ImVec2(0,0), int flags = -1);
bool StyledSmallButton(const char* label, int flags = -1);   // small = no vertical pad

Rendered as a full pill (rounding = height*0.5, ignores FrameRounding): an OKLAB fill crossfade (ImGuiCol_Button/Hovered/Active), a soft hover glow, a top gloss, an always-visible theme-accent rim, a spring press scale (shrinks while held, overshoots past 1.0 on release), and a diagonal hover sheen masked to the pill. The fill is forced near-opaque and the label gets a drop shadow so the effects never wash the text. kDefaultButtonFlags = GlowOnHover | PressSink | Shimmer. The toolbar's IconButton is built from the same primitives (rounded-square slots instead of pills).

Edit popups & click-to-edit field

One shared cursor-anchored popup body, EditPopupRun(popupId, width, extraFlags, hint), backs both the slider's Ctrl+click numeric editor and the generic input popup: a focused InputText + styled Accept/Cancel, Enter accepts / Esc cancels, one shared scratch buffer (one popup open at a time). The shared file-static ImGuiID g_editPopupOwner (Widgets.cpp:702) records which control opened the popup; EditPopupRun closes and early-returns if the open popup is not owned by the caller, preventing the slider edit and generic InputPopup from stomping each other's shared g_editPopupBuf.

// Generic input popup (open at the mouse, then poll each frame)
void OpenInputPopup(const char* popupId, const char* initialText);
bool InputPopup(const char* popupId, std::string& out, const char* hint = nullptr);

// Click-to-edit field: read-only framed display + pencil; click opens the popup.
bool StyledEditField(const char* label, std::string* value, const char* hint = nullptr);
bool StyledEditField(const char* label, char* buf, size_t bufSize, const char* hint = nullptr);

StyledEditField draws a rounded read-only display of the value (with an ICON_FA_PENCIL affordance and hover-eased bg/rim), and on click opens InputPopup to edit — writing the value and returning true only on accept. It's the drop-in for persistent value fields you don't want saving per keystroke (per-window string settings, the bar Text format, the ThemeZ name). Live search/filter boxes and type-then-press add fields stay as plain ImGui::InputText.
