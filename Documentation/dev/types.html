Anonymizer (name masking)

When mq::IsAnonymized() is active, every displayed character name and tooltip is replaced with a RACE_CLASS_LEVEL code (e.g. HIE_WIZ_65) — a string that can never be a real name, so a stream can't be scraped to mass-report players. Self renders as "Me". Helpers: UiHelpers.h RaceShort(raceId) + MaskedCode(raceId, classShort, level). Masking is local-only — actor messages carry the real name (used for commands / selection keys); the peer's real race rides the vitals message so each client builds the code locally (PeerRecord.maskedName). PC spawn targets build the code inline (NPCs keep their name); name-only spots (buff caster, group-leader header) fall back to a generic token; display sites use a ##realname id so duplicate codes don't collide.

Build gotchas (compile-time)

  • TUs dereferencing a ForeignPointer global (pAAWnd, pAltAdvManager, pLocalPC, pDBStr, pCastSpellWnd, pItemDisplayManager, pSpellDisplayManager) need #include <cassert>.
  • Item-spell enum type is ItemSpellTypes (alias eItemSpellType), NOT ItemSpellType.
  • VePointer has only an explicit operator bool — use if/ternary/ static_cast<bool>, not != nullptr.
  • mq::join does not exist in this MQ install (only split in mq/base/String.h).
  • Things not pulled by <mq/Plugin.h> need explicit includes (mq/imgui/Widgets.h, imgui/fonts/IconsFontAwesome.h, mq/base/String.h).
  • A SpanAllColumns Selectable needs ImGuiSelectableFlags_AllowOverlap or it eats clicks of buttons in other columns.
