Module System

Every window in MQMyUI is a ModuleBase subclass. The system is deliberately small: an abstract base with five virtual hooks, a context struct of shared service pointers, and a manager that owns the modules and drives their lifecycle. This page is the contract you implement when adding a window.

ModuleBase (Core/ModuleBase.h:25)

The base declares the lifecycle hooks (all no-ops by default except the pure-virtual GetName()) and holds the injected ModuleContext.

class ModuleBase
{
public:
    virtual const char* GetName() const = 0;   // window + config key, also /myui <name>

    virtual void OnInit() {}        // once, when the character goes in-game
    virtual void OnPulse() {}       // throttled background work (~250ms)
    virtual void OnRenderGUI() {}   // every frame; do all ImGui here
    virtual void OnShutdown() {}    // teardown on logout / plugin unload
    virtual void OnZone() {}        // on zone change

    bool IsEnabled = true;          // disabled modules skip Init/Pulse/Render/Zone

protected:
    ModuleContext m_ctx;            // shared services, injected by the manager
};
Hook discipline: do all ImGui drawing in OnRenderGUI only. Use OnPulse for periodic non-UI work (e.g. rebuilding a derived list) — it runs about every 250 ms, not every frame. Never poll the game's TLOs in render; read m_ctx.Char instead.

ModuleContext (Core/ModuleBase.h:14)

A flat struct of raw pointers to the seven service singletons. The manager sets it on each module via the friended SetContext before OnInit. Modules reach everything through m_ctx.

FieldTypeUse it for
SettingsSettingsStore*Raw SQLite key/value + theme rows (usually go through UI instead)
ThemesThemeManager*The active ImGui theme (Themes editor only)
CharCharData*Local character snapshot: vitals, buffs, group, raid
ChatChatBridge*Routed chat output (MQMyChat or console)
ActorsActorManager*Peer registry + remote commands
IconsIconHelper*Spell / item / status icon drawing
UIUiConfig*This window's visibility, scaling, flags, and bar styles

ModuleManager (Core/ModuleManager.h:9)

Owns the modules in a vector and drives them.

MethodDoes
RegisterModule<T>()Constructs T, injects the context, stores it. Call order = render order.
SetContext(ctx)Stores the context to inject into every registered module.
InitAll / PulseAll / RenderAll / ZoneAllIterate enabled modules, calling the matching hook.
ShutdownAllIterates all modules (even disabled) for clean teardown.
Find(name) / SetEnabled(name, on)Case-insensitive lookup and runtime enable toggle.

A minimal module

The smallest real module, ITrackModule, shows the shape: a header declaring GetName() and the one hook it overrides, plus private runtime state.

// Modules/ITrackModule.h
class ITrackModule : public ModuleBase
{
public:
    const char* GetName() const override { return "iTrack"; }
    void OnRenderGUI() override;

private:
    void DrawItemList();
    void DrawItemDetails();

    char m_input[128] = { 0 };   // transient UI state only — never settings
    std::string m_selected;
};

Adding a new window

  1. Create Modules/MyThingModule.h/.cpp subclassing ModuleBase; return a unique key from GetName() (e.g. "MyThing").
  2. Render in OnRenderGUI. Read WindowConfig& w = m_ctx.UI->Window(GetName());, bail if !w.visible, pass &w.visible to ImGui::Begin so the X-close flips visibility, and persist visibility changes immediately (see Settings & UiConfig).
  3. Seed defaults for the window in UiConfig::SeedDefaults (visibility, flags, nums, bar roles). Without a seed entry the window still works but starts with bare defaults.
  4. Register it in RegisterModules() (MQMyUI.cpp) at the position that gives the render order you want.
  5. Add the files to MQMyUI.vcxproj and .vcxproj.filters.
The window automatically gains a /myui MyThing toggle and shows up in /myui list, because both resolve against UiConfig::WindowKeys() — no extra wiring.
A module's private members are for transient runtime state only (selection, scratch buffers, animation timers). Anything the user should keep across relogs belongs in UiConfig / SettingsStore, read live each frame — not in a member loaded once in OnInit.
